Policy Layer For Agentic Coding

Let agents move fast without letting them touch everything.

agent-guard sits between coding agents and risky actions. It reads normalized events, matches them against policy packs, and returns an explicit decision: allow, warn, or block.

The point is simple: the model should not be the only thing deciding whether a command, edit, or production access is acceptable.

01

Normalize runtime noise

Claude Code hooks, Codex wrappers, CI jobs, and MCP gateways can all emit different payloads. agent-guard adapters collapse them into one policy shape.

02

Evaluate with portable rules

Rules are plain YAML or JSON. Match on shell commands, file paths, tags, prompt text, MCP targets, or custom metadata fields.

03

Return a hard decision

Every event ends in a deterministic result. No vague “maybe risky” output. Your wrapper or workflow gets `allow`, `warn`, or `block`.
